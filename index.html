Getting Started

Using this service will reduce the image payload significantly and increase the website’s performance.

The easiest way to understand this service is to see it as an intelligent Content Delivery Network (CDN). The built-in intelligence automatically (or explicitly) optimizes images for the requesting device or browser. In most cases the data traffic generated by images will be reduced up to 90 percent compared to the original images.

Basic Images Transformations

Go to your compression server URL, for example https://compress.id.net.co then append /upload/; after that you can pass these options below, followed by an underscore and a value q_75. Options are separated by coma (configurable to other separator).

After the options put the source of your image, it can be relative to your server or absolute, for example: /https://yourwebsite.com/path/your-images.jpg

So to get a pretty dog a with 75% compression, you would write: https://compress.id.net.co/upload/q_75/https://yourwebsite.com/path/your-images.jpg

example: https://compress.id.net.co/upload/q_75/https://img.id.net.co/wp-content/uploads/2021/07/newborn-6467761_640.jpeg

Image Manipulation Options

w : width

Default: null

Description: Sets the target width of the image. If not set, width will be calculated in order to keep aspect ratio.

example: w_250

w_250 : https://compress.id.net.co/upload/w_250/https://img.id.net.co/wp-content/uploads/2021/07/newborn-6467761_640.jpeg

h : height

Default: null

Description: Sets the target height of the image. If not set, height will be calculated in order to keep aspect ratio.

example: h_250

h_250 : https://compress.id.net.co/upload/h_250/https://img.id.net.co/wp-content/uploads/2021/07/newborn-6467761_640.jpeg

Using width AND height

example: h_250,w_250

By default setting width and height together, works like defining a rectangle that will define a max-width and max-height and the image will scale proportionally to fit that area without cropping.

By default; width, height, or both will not scale up an image that is smaller than the defined dimensions.

h_250,w_250 : https://compress.id.net.co/upload/h_250,w_250/https://img.id.net.co/wp-content/uploads/2021/07/newborn-6467761_640.jpeg

c : crop

Default: false

Description: When both width and height are set, this allows the image to be cropped so it fills the width x height area.

example: c_1

c_1,h_250,w_250 : https://compress.id.net.co/upload/c_1,h_250,w_250/https://img.id.net.co/wp-content/uploads/2021/07/newborn-6467761_640.jpeg

smc : smart crop

Default: false

Description: Smart cropping feature, uses python script to determine coordinates.

example: smc_1

smc_1,w_250 : https://compress.id.net.co/upload/smc_1,w_300/https://img.id.net.co/wp-content/uploads/2021/07/newborn-6467761_640.jpeg

g : gravity

Default: center

Description: When crop is applied, changing the gravity will define which part of the image is kept inside the crop area. The basic options are: NorthWest, North, NorthEast, West, Center, East, SouthWest, South, SouthEast

example: g_West

g_West : https://compress.id.net.co/upload/c_1,g_West,h_300,w_300/https://img.id.net.co/wp-content/uploads/2021/07/newborn-6467761_640.jpeg

r : rotate

Default: null

Description: Apply image rotation (using shear operations) to the image.

example: r_90, r_-180, ….

r_45 : https://compress.id.net.co/upload/r_45,h_300,w_300/https://img.id.net.co/wp-content/uploads/2021/07/newborn-6467761_640.jpeg

o : output

Default: auto

Description: Output format requested, for example you can force the output as jpeg file in case of source file is png. The default auto will try to output the best format for the requesting browser, falling back to the same format as the source image or finally with a fallback to jpg.

If input is passed, no “optimal” format will be attempted. The service will try to respond with the source format or fallback to jpg.

example: o_auto,o_input,o_png,o_webp,o_jpeg,o_jpg

o_auto : https://compress.id.net.co/upload/o_auto/https://img.id.net.co/wp-content/uploads/2021/07/newborn-6467761_640.jpeg

q : quality

Default: 90

Description: Sets the compression level for the output image. Your best results will be between 70 and 95.

example: q_95, q_75, …

q_75 : https://compress.id.net.co/upload/q_75/https://img.id.net.co/wp-content/uploads/2021/07/newborn-6467761_640.jpeg

webpl : webp-loseless

Default: 0

Description: If output is set to webp, it will default to lossy compression, but if you want lossless webp encoding you have to set it to 1.

example: webpl_1

webpl_1 : https://compress.id.net.co/upload/q_75,webpl_1/https://img.id.net.co/wp-content/uploads/2021/07/newborn-6467761_640.jpeg

rf : refresh

Default: false

Description: When this parameter is 1, it will force a re-request of the original image and run it through the transformations and compression again. It will delete the local cached copy.

The nginx server will send headers to prevent caching of this request.

It will also send headers with the command done on the image + info returned by the command identity from Imagemagik.

example: rf_1

rf_1 : https://compress.id.net.co/upload/q_75,rf_1/https://img.id.net.co/wp-content/uploads/2021/07/newborn-6467761_640.jpeg

bg : background

color (multiple formats)

Default: null

Description: Sets the background of the canvas for the cases where padding is added to the images. It supports hex, css color names, rgb. Only css color names are supported without quotation marks. For the hex code, the hash # character should be replaced by %23.

example: bg_yellow, bg_%23ff4455, bg_rgb(255,120,100), …

bg_yellow : https://compress.id.net.co/upload/q_75,bg_yellow,r_45/https://img.id.net.co/wp-content/uploads/2021/07/newborn-6467761_640.jpeg

st : strip

Default: 1

Description: removes exif data and additional color profile. Leaving your image with the default sRGB color profile.

example: st_1

rz : resize

Default: null

Description: The alternative resizing method to -thumbnail.

example: rz_1

rz_1 : https://compress.id.net.co/upload/rz_1/https://img.id.net.co/wp-content/uploads/2021/07/newborn-6467761_640.jpeg

moz : mozjpeg

Default: 1

Description: Use moz-jpeg compression library, if 0 it fallsback to the default ImageMagick compression algorithm.

unsh : unsharp

{radius}x{sigma}[+gain][+threshold]

Default: null

Description: Sharpens an image (despite the unfortunate naming). It basically sharpens via subtracting a blurred, low contrast, version of the image to the image itself. For more details check Imagemagick docs.

example 1: unsh_0x6
example 2: unsh_0.25×0.25+8+0.065

unsh_0x6 : https://compress.id.net.co/upload/unsh_0x6/https://img.id.net.co/wp-content/uploads/2021/07/newborn-6467761_640.jpeg

sh : sharpen

{radius}x{sigma}

Default: null

Description: Use a Gaussian operator of the given radius and standard deviation (sigma). For more details check Imagemagick docs.

example 1: sh_0x6
example 2: sh_0.25×0.25+8+0.065

sh_0x6 : https://compress.id.net.co/upload/sh_0x6/https://img.id.net.co/wp-content/uploads/2021/07/newborn-6467761_640.jpeg

blr : blur

{radius}x{sigma}

Default: null Description: Apply Blur on a image. For more details check Imagemagick docs.

example 1: blr_2
example 2: blr_1x2

blr_2 : https://compress.id.net.co/upload/blr_2/https://img.id.net.co/wp-content/uploads/2021/07/newborn-6467761_640.jpeg

f : filter

Default: Lanczos

Description: Resizing algorithm, Triangle is a smoother lighter option.

example: f_Triangle

sc : scale

Default: Null

Description: The “-scale” resize operator is a simplified, faster form of the resize command. Useful for fast exact scaling of pixels.

example: sc_1

PDF Options

pg : page number

Default: 1

Description: Sets the target page of the PDF. If not set, the default is page 1.

example: pg_3

pg_3 : https://compress.id.net.co/upload/pg_3/https://img.id.net.co/wp-content/uploads/2021/07/c4611_sample_explain.pdf

Video Options

tm : time

Default: 00:00:01

Description: Sets the frame capture time duration point in the video. If not set, the default is 1 second. The format is HH:MM:SS OR SS.

example: tm_00:00:05

tm_00:00:05 : https://compress.id.net.co/upload/tm_00:00:05/https://img.id.net.co/wp-content/uploads/2021/07/file_example_MP4_640_3MG.mp4

WordPress Integrations

1. CDN Enabler Plugin

Enabling the service for your WordPress image content is very simple. You can use plugin such as CDN Enabler (free).

NOTE: You must use CDN Enabler version 1.0.9 to make it work with the service. Download here.

CDN URL: Insert endpoint URL and your URL, for example https://compress.id.net.co/upload/q_75/https://your-website.com

Included Directories: wp-content

Exclusions: .php,.js,.css,.eot,.woff,.woff2,.ttf,.svg, .zip,.ico, .mp4, .mp3

Relative Path: check

CDN HTTPS: check
